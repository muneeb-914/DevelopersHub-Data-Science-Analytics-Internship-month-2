# Task 4: Loan Default Risk with Business Cost Optimization

## Objective
Predict loan default risk with binary classification models, then choose the lending decision threshold using a business cost framework instead of accuracy alone.

## Dataset
This project uses `data/application_train.csv`, the Home Credit Default Risk training dataset.

Verified local dataset facts:

| Item | Value |
|---|---:|
| Rows | 307,511 |
| Raw columns | 122 |
| Repaid loans (`TARGET=0`) | 282,686 |
| Defaulted loans (`TARGET=1`) | 24,825 |
| Default rate | 8.07% |

`TARGET` is the binary label: `1` means defaulted and `0` means repaid.

## Approach
1. Load `application_train.csv` and inspect class imbalance, missing values, numeric distributions, and target correlations.
2. Engineer features:
   - `AGE_YEARS` from `DAYS_BIRTH`
   - `EMPLOYMENT_YEARS` from `DAYS_EMPLOYED` after replacing the Home Credit `365243` sentinel with missing
   - `CREDIT_INCOME_RATIO`
   - `ANNUITY_INCOME_RATIO`
3. Label-encode categorical columns for the baseline model workflow.
4. Use a stratified 80/20 train-test split.
5. Impute missing values with medians learned from the training split.
6. Train:
   - Logistic Regression with `class_weight='balanced'`
   - CatBoost with `scale_pos_weight`
7. Sweep thresholds from 0.01 to 0.99 and select the threshold with the lowest test-set business cost.

## Business Cost Matrix
| Scenario | Model Prediction | Reality | Cost |
|---|---|---|---:|
| Approve applicant who defaults | Repaid / negative | Default / positive | $25,000 |
| Reject applicant who would repay | Default / positive | Repaid / negative | $5,000 |
| Approve applicant who repays | Repaid / negative | Repaid / negative | $0 |
| Reject applicant who defaults | Default / positive | Default / positive | $0 |

The notebook computes total cost as:

```text
total_cost = false_negatives * 25000 + false_positives * 5000
```

## Results
The notebook generates the current run's ROC-AUC, average precision, optimal thresholds, confusion matrices, total costs, savings, and CatBoost feature importance.

Exact metric values are not hard-coded in this README because they should come from the latest notebook execution against the current CSV and package versions.

## Business Recommendations
- Choose the model-threshold pair with the lowest held-out test-set business cost.
- Do not rely on the default `0.5` threshold without comparing it to the cost-optimal threshold.
- Re-run threshold optimization when portfolio risk, approval policy, or cost assumptions change.
- Monitor the data quality of the features that appear in the generated feature-importance output.
- Consider product-level or segment-level thresholds only after validating enough holdout samples per segment.
